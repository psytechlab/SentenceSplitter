# Agent Annotator — многомодельный эксперимент

## Цель

Измерить, насколько LLM-агент способен заменить / дополнить человека-разметчика на задаче разбиения сложных русских предложений на клаузы.  
Сравниваются несколько моделей (Sonnet 4.6, Opus 4.8) против разметки трёх аннотаторов (Даниил, Ширин, Игорь) на 200-текстовом валидационном сете.  
**Финальный вывод:** `comparison.md`

---

## Архитектура агента

| Параметр | Значение |
|---|---|
| Среда | Claude Code |
| Модель | claude-sonnet-4-6 (Claude Sonnet 4.6) |
| Claude Code версия | 2.1.161 |
| Batch size | 20 текстов за раз |
| Батчей всего | 10 |
| Concurrency | 1 (батчи последовательно) |
| Temperature | default |
| Retry policy | до 2 retry на батч; fallback single-turn не потребовался |

**Системный промпт (единый на весь прогон):**
- `instructions_snapshot.md` — полная инструкция Ширин (зафиксирована 2026-06-03)
- `few_shot_examples.md` — 6 эталонных разметок (единогласное согласие всех трёх аннотаторов)
- `checklist.md` — 12 правил «если видишь X — делай Y»

**Формат output батча:** JSON-массив из 20 объектов `{task_id, spans, sentence_type, reasoning}`

## Вывод

### Достаточно ли для **замены человека**?

**Нет** по токен-alpha и F1 — заметный gap с лучшим человеком (Shirin κ=0.971 vs агент 0.899). Агент по 5-class granularity иногда превышает межчеловеческий IAA (κ агент–Daniil=0.802 > Daniil–Igor=0.742), но это объясняется тем, что агент ближе к Daniil'ю по паттернам разметки.

### Достаточно ли для **LLM-assisted** (аннотатор редактирует)?

**Да.** Token α 0.843, boundary F1 relaxed 0.854, 5-class κ 0.712–0.802 — всё выше порога 0.7 и в пределах разброса людей. ~95% текстов агент аннотирует верно по бинарному критерию, а разметка spans достаточно близка к человеческой для редактирования.

### Есть ли контекстный дрейф?

**Да, умеренный.** F1 в батчах 6–9 (~0.79–0.80) ниже, чем в батчах 1–5 (~0.90). Батч 8 — минимум Token α (0.753). Однако батчи 6–9 также содержат объективно сложные тексты потока сознания (presuicidal_multilabel), поэтому нельзя однозначно атрибутировать ухудшение контекстному дрейфу. Для разделения рекомендуется single-turn контрольный прогон на батчах 6–9.

### Категории для доработки промпта

1. **Однородные предикаты при одном подлежащем** (ошибки A) — подчеркнуть правило 2.6.1, добавить примеры фазовых глаголов ('стала + инфинитив') и апозитивов через тире
2. **Уточняющие обороты без глагола** (ошибки A/F) — добавить правило: «нет сказуемого = нет клаузы, даже при запятой»
3. **BSP в коротких текстах без пунктуации** (ошибки B) — добавить в чеклист: «ищи предикат невзирая на смайлы и отсутствие знаков»
4. **Вставная конструкция 'как X сказал/говорит'** (ошибки E) — добавить явный пример выделения
5. **Длинные тексты** (ошибки D) — добавить инструкцию: «для текстов >300 символов делай более тонкое разбиение»

## Известные ограничения

1. **Batch=20 vs single-turn**: batch-режим потенциально вносит контекстный дрейф. Качество single-turn может быть выше (но в 20× дороже).
2. **Одна модель**: использован только claude-sonnet-4-6. Сравнение с claude-opus-4-8 требует повторного прогона с переключением модели.
3. **Агент ≠ API**: разметка выполнена через Claude Code в интерактивном режиме, не через чистый API-вызов. Реальный прогон через API может давать иные числа.
4. **Нет «золотых» spans**: majority GT spans не существует (только majority_type). Span-метрики считаются как агент vs каждый аннотатор, не vs агрегат.
5. **198 из 200**: 2 задачи с null stype у кого-то из людей исключены из 5-class анализа.
