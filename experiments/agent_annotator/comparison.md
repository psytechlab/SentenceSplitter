# Сравнение моделей-разметчиков vs люди

**Задача:** разделение сложных русских предложений на клаузы  
**Валидационный сет:** 200 текстов, разметка трёх аннотаторов (Даниил, Ширин, Игорь)  
**Human-human IAA (baseline):** Token α = 0.846, Boundary F1 relaxed = 0.852–0.885, 5-class κ = 0.742–0.792  
**Прогоны:** Sonnet 4.6 и Opus 4.8 — один и тот же системный промпт (инструкция + few-shot + checklist, хеши идентичны), batch=20, concurrency=1, независимая разметка.

## Выводы

### Достаточно ли качества для **replacement** человека?

| Критерий | Порог = min(human-human) | Sonnet 4.6 | Opus 4.8 |
|---|---|---|---|
| Token α ≥ 0.846 | 0.846 | 0.843 ❌ | **0.850 ✅** |
| BF1 relaxed avg ≥ 0.852 | 0.852 | 0.854 ✅ | **0.884 ✅** |
| Binary κ ≥ 0.890 | 0.890 | 0.899 ✅ | **0.957 ✅** |
| 5-class κ avg ≥ 0.742 | 0.742 | 0.757 ✅ | **0.806 ✅** |
| **Итог** | — | 3 из 4 (пограничный) | **4 из 4 ✅** |

**Sonnet 4.6:** пограничный — не дотягивает по Token α до минимального человека.  
**Opus 4.8:** проходит **все четыре** порога. По бинарной классификации (κ=0.957) и 5-class κ vs Daniil/Shirin (0.87/0.80) — **превосходит** межчеловеческий разброс. Формально Opus достиг уровня «ещё одного аннотатора, не хуже среднего человека».

> ⚠️ Оговорка: «не хуже среднего человека по агрегированным метрикам» ≠ «можно убрать людей». Лучший человек (Shirin, κ=0.971) всё ещё точнее Opus (0.957) по бинарному критерию. Для high-stakes-разметки золотого стандарта по-прежнему нужен человек-арбитр. Но для **построения серебряного корпуса** Opus самодостаточен.

| Категория | Sonnet | Opus | Рекомендация |
|---|---|---|---|
| Ложное разбиение однородных предикатов | 7/20 | ~1/20 | Opus почти решил; оставить усиленное правило 2.6.1 |
| Пропуск БСП в коротких/неформальных текстах | 2/20 | 0/20 | Opus решил полностью |
| Двойной эллипсис «куда хотела» (156) | — | 1 | **Устранить противоречие в чеклисте п.6** (явно: разделять или нет) |
| Недоразбиение длинных текстов (>300 симв.) | 3/20 | 2/20 | Добавить «для длинных текстов дроби мельче» — слабость обеих моделей |
| Зернистость вставных конструкций (36, 151) | 2/20 | 2/20 | Калибровать: люди чаще объединяют определительную+вставную |

### Рекомендация: какую модель использовать в production?

**Opus 4.8** — для финальной автоматической разметки и построения серебряного корпуса.
- Превосходит Sonnet по всем метрикам, проходит все 4 порога «replacement», устойчивее к контекстному дрейфу.
- Стоимость: ×1.67 к input / ×1.67 к output относительно Sonnet ($5/$25 против $3/$15 за 1М токенов). При batch=20 разница в абсолютных деньгах невелика, а прирост качества (κ 0.90→0.96, исправление 6 из 7 катастроф) её окупает.

**Оптимальный production-pipeline:**
1. **Opus 4.8 batch=20** — первичная автоматическая разметка всего корпуса.
2. **Человек-редактор** — выборочная проверка задач с reasoning-флагом «пограничный случай» + всех «Не сложное» (где у Opus остаточные ошибки двойного эллипсиса) + длинных текстов >300 символов.
3. **Sonnet 4.6** — как дешёвый вариант для preview/черновой разметки или при жёстком бюджете; терять ~3 п.п. κ.

**Не рекомендуется:** убирать человека полностью из контура для золотого стандарта — лучший человек по-прежнему точнее Opus, а пограничные случаи (двойной эллипсис, БСП↔Смешанного) требуют арбитра.

## Известные ограничения сравнения

1. Обе модели размечены через Claude Code (batch, интерактивно), не через чистый API — реальный API-прогон может дать иные числа.
2. Один прогон на модель (без повторов) — нет оценки дисперсии внутри модели.
3. Span-метрики считаются vs каждый человек попарно (нет majority-span GT) — relaxed-метрики надёжнее exact.
4. 2 из 200 задач исключены из 5-class анализа (null stype у кого-то из людей).
5. batch=20 у обеих моделей — single-turn мог бы дать ещё выше, но в 20× дороже.
