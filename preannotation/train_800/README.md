# Предразметка train_800 — Opus 4.8

Автоматическая («серебряная») разметка 800 текстов
`data/sample_for_annotation_train_800.csv` по задаче «разделение сложного
предложения на клаузы», выполненная **Claude Opus 4.8** — лучшей моделью по
итогам валидационного эксперимента (`experiments/agent_annotator/comparison.md`).

Предназначена для **LLM-assisted разметки**: человек-редактор проверяет и
правит предсказания, а не размечает с нуля.

## Модель и контекст

- Модель: **claude-opus-4-8** (Claude Opus 4.8), через Claude Code, batch=20 (40 батчей)
- Системный контекст — тот же, что в валидационном эксперименте:
  `experiments/agent_annotator/prompt/` (инструкция Ширин + few-shot + checklist)
- Разметка независимая, по собственному суждению модели; нет пересечения с
  размеченными 200 валидационными текстами.

## Файлы

| Файл | Назначение |
|---|---|
| `inputs.json` | 800 входных текстов с `task_id` (0–799), `data_id`, `stratum` |
| `annotations.json` | основной результат: `task_id`, `data_id`, `stratum`, `text`, `spans` (символьные смещения), `sentence_type`, `reasoning` |
| `annotations.csv` | плоская версия (по строке на текст; `spans_json` сериализован) |
| `label_studio_predictions.json` | импорт в Label Studio: разметка лежит в `predictions[]` (`model_version: opus-4.8-preannotation`) — человек открывает и правит |

## Формат разметки

- `spans` — список клауз: `{start, end, text}` (символьные смещения в `text`, exclusive end).
- `sentence_type` — один из: `Не сложное`, `ССП`, `СПП`, `БСП`, `Смешанного типа`.
- `reasoning` — 1–2 фразы обоснования.

## Сводная статистика (800 текстов)

- Типы: Смешанного типа 313, Не сложное 182, БСП 153, СПП 125, ССП 27
- Бинарно: сложных 618, простых 182
- Средняя зернистость: 3.08 спана на текст

Распределение по стратам согласуется с валидационным сетом (короткие твиты и
`presuicidal_short_simple` — преимущественно простые; длинные страты — сложные).

## Оговорки (качество серебряной разметки)

По метрикам на валидации Opus 4.8 ≈ уровень человека-аннотатора (Token α 0.85,
binary κ 0.96), но **не безошибочен**. Приоритет для ручной проверки:
1. Тексты `Не сложное` (остаточные ошибки двойного эллипсиса «куда хотела»).
2. Длинные тексты потока сознания (>300 символов) — возможно недоразбиение.
3. Вставные конструкции и границы БСП/Смешанного.

## Воспроизведение

1. Восстановить `inputs.json` из `data/sample_for_annotation_train_800.csv`
   (порядок строк CSV = `task_id` 0–799).
2. В Claude Code выбрать модель `claude-opus-4-8`, системный промпт —
   `experiments/agent_annotator/prompt/{instructions_snapshot,few_shot_examples,checklist}.md`.
3. Разметить батчами по 20, склеить в `annotations.json`.

Окружение зафиксировано в `env.txt`.
